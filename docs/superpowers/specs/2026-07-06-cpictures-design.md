# cpictures 图片查看器设计规格

日期：2026-07-06  
平台：Windows 11  
技术路线：C++20 + Win32 + WIC + Direct2D/DXGI + 扩展解码插件

## 目标

从零开发一款 Win11 图片查看器，核心目标是超快、超轻、极简。

主要用户：
- 日常用户：快速打开、切换、缩放、复制图片文件。
- 设计师：查看 PNG、WebP、AVIF、SVG、PSD/PSB、EXR、HDR。
- 摄影师：查看 JPEG、TIFF、HEIC、RAW（CR2/CR3/NEF/ARW/DNG/RW2/ORF/RAF）。

成功标准：
- 双击图片后快速显示，主程序本体尽量小。
- 窗口默认只显示图片本身，无标题栏、无边框、无工具条。
- 默认 100% 原图显示；窗口自动贴合图片尺寸，最大不超过屏幕。
- 同目录图片自动组成浏览列表，滚轮切换前后图片。
- 常用功能通过右键菜单和快捷键完成。
- 可注册为 Win11 默认图片应用，遵守系统默认应用机制，不静默抢占关联。

## 非目标

第一版不做：
- 图片编辑。
- 图库管理。
- 删除到回收站。
- 打印。
- 文件移动、另存为、批处理。
- 常驻工具栏、标题栏、状态栏。
- 静默安装或静默更新第三方组件。

## 用户体验

### 窗口

图片查看窗口默认无标题栏、无边框、无额外控件，只绘制图片本身。

打开图片时：
1. 解码图片并读取像素尺寸。
2. 若图片 100% 尺寸小于当前显示器工作区，窗口客户区等于图片像素尺寸。
3. 若图片 100% 尺寸超过当前显示器工作区，窗口缩放到工作区内可见范围，保持宽高比。
4. 窗口居中显示在当前显示器。

窗口行为：
- `Esc` 关闭窗口。
- 单击图片任意区域，切换左下角信息角标显示/隐藏。
- 角标默认隐藏。
- 角标内容格式：`文件名.格式  n/m`，例如 `DSC_1042.NEF  17/128`。
- 角标位置：图片左下角。
- 角标样式：黑色背景 34% 不透明度，白字 90%，小字号，轻微圆角。

### 鼠标

- 滚轮：切换上一张/下一张。
- `Ctrl + 滚轮`：缩放。
- 左键短按单击：显示/隐藏左下角信息角标。
- 右键：打开上下文菜单。
- 左键按住并超过拖动阈值：移动窗口。由于窗口无标题栏，采用命中测试模拟标题栏拖动。
- 图片放大并可平移时：使用中键拖动或空格+左键拖动平移图片，避免和移动窗口冲突。

### 键盘

- `Esc`：关闭。
- `←` / `PageUp`：上一张。
- `→` / `PageDown`：下一张。
- `Ctrl + 滚轮` / `+` / `-`：缩放。
- `1`：实际大小（100%）。
- `0`：适应屏幕。
- `F11`：全屏切换。
- `Ctrl + L`：向左旋转（临时显示，不写回文件）。
- `Ctrl + R`：向右旋转（临时显示，不写回文件）。
- `Ctrl + C`：复制文件，行为对齐资源管理器复制文件。
- `Ctrl + Shift + C`：复制完整路径。

### 右键菜单

右键菜单承载所有功能和设置入口，保持查看窗口极简。

菜单项：
- 向左旋转
- 向右旋转
- 复制
- 复制路径
- 实际大小
- 适应屏幕
- 全屏
- 安装/更新格式支持...

不包含：
- 删除
- 在资源管理器中显示
- 打开方式
- 属性
- 复制图片内容

`复制` 使用 Windows Shell 剪贴板文件格式（`CF_HDROP` 等），语义对齐资源管理器：粘贴到文件夹时复制原文件。

`复制路径` 写入文本剪贴板，内容为当前图片完整路径。

## 图片列表和切换

打开任意图片后，程序扫描同目录支持格式图片，生成浏览列表。

排序：
- 使用自然排序，例如 `2.jpg` 排在 `10.jpg` 前。
- 默认按文件名升序。

性能：
- 当前图片同步加载。
- 后台轻量预读前后各 1 张。
- 预读受内存上限控制，大图优先保存缩略/解码句柄，不无限缓存全尺寸位图。
- 切换图片时更新窗口尺寸策略：默认仍以当前图片 100% 尺寸贴合，最大不超过屏幕。

## 格式支持

### 核心支持

核心程序优先使用 Windows Imaging Component（WIC）和系统已安装编解码器。

核心目标格式：
- JPEG/JPG
- PNG
- BMP
- GIF
- TIFF/TIF
- WebP（系统支持时）
- HEIC/HEIF（系统支持时）
- ICO（系统支持时）

### 扩展支持

采用“原生核心 + 扩展解码插件”方案。

扩展格式优先级：
- AVIF
- SVG
- PSD/PSB
- RAW：CR2、CR3、NEF、ARW、DNG、RW2、ORF、RAF
- EXR
- HDR

扩展插件按需加载：
- 主程序启动不加载扩展 DLL。
- 打开对应格式时再加载对应解码插件。
- 插件失败不影响主程序启动和核心格式查看。

### 动图

第一版支持 GIF/WebP 动画的基础播放。若解码器只返回静态首帧，则显示首帧并保持稳定，不阻塞查看流程。

### EXIF 和色彩

- 自动应用 EXIF Orientation。
- 尽量保留 ICC 色彩配置；WIC/Direct2D 路线可用系统色彩管理能力。
- 临时旋转只改变显示矩阵，不修改原文件。

## 扩展组件下载

支持两种入口：
1. 打开不支持格式时提示下载对应组件。
2. 右键菜单点击 `安装/更新格式支持...` 手动检查和更新。

下载流程：
1. 请求远端 `manifest.json`。
2. 展示组件名、版本、大小、来源和支持格式。
3. 用户确认后下载。
4. 校验 SHA-256。
5. 解压/写入本地 `codecs\` 目录。
6. 更新本地组件清单。
7. 重新尝试打开图片或提示重试。

安全原则：
- 不静默下载。
- 不加载 hash 校验失败的 DLL。
- 不因扩展下载失败影响核心查看能力。
- 扩展组件版本、hash、来源记录在本地清单。

## 文件关联和安装

安装器注册应用能力，让 Win11 默认应用页面可选择 cpictures。

要求：
- 注册支持扩展名和 ProgID。
- 支持“打开方式”。
- 支持用户在系统设置中设为默认图片应用。
- 不使用非正规方式静默抢占默认应用。

可提供两类产物：
- 安装包：用于注册默认应用和扩展组件目录。
- 便携 exe：可直接打开图片，但不保证默认应用注册。

## 架构

### 模块

- `App`：入口、命令行解析、单实例或多实例策略。
- `ViewerWindow`：Win32 窗口、消息循环、DPI、无边框命中测试、输入处理。
- `Renderer`：Direct2D/DXGI 渲染、缩放、旋转、全屏。
- `ImageDocument`：当前图片状态、尺寸、EXIF、ICC、帧信息。
- `ImageList`：目录扫描、自然排序、当前索引、前后切换。
- `DecodeManager`：统一解码接口，先 WIC 后扩展插件。
- `CodecPlugin`：扩展解码插件 ABI。
- `Prefetcher`：后台预读前后 1 张。
- `ClipboardService`：复制文件、复制路径。
- `ContextMenu`：右键菜单构建和命令分发。
- `UpdateService`：扩展组件 manifest、下载、hash 校验、安装。
- `Settings`：少量状态持久化，例如扩展目录、角标默认隐藏、最近窗口行为。

### 解码流

1. 用户打开文件路径。
2. `ImageList` 扫描同目录，定位当前索引。
3. `DecodeManager` 尝试 WIC 解码。
4. 若 WIC 不支持，检查本地扩展插件。
5. 若仍不支持，触发按需下载提示。
6. 解码成功后生成 `ImageDocument`。
7. `ViewerWindow` 按尺寸策略调整窗口。
8. `Renderer` 绘制图片。
9. `Prefetcher` 后台预读前后图片。

### 插件 ABI

插件 API 保持 C ABI，便于 C++ 主程序稳定加载：
- 查询插件信息：名称、版本、支持扩展名、能力。
- 探测文件：是否可解码。
- 解码文件：输出像素缓冲、尺寸、像素格式、色彩信息、帧信息。
- 释放资源。

主程序负责：
- DLL 加载和卸载。
- 线程隔离。
- 超时和错误处理。
- 将插件输出转为 Direct2D 可用位图。

## 错误处理

- 文件不存在：显示轻量错误提示窗口或消息框，说明路径不可用。
- 格式不支持：提示安装对应格式支持，或说明当前无法打开。
- 解码失败：显示“无法解码此图片”，保留前后切换能力。
- 扩展下载失败：说明网络或校验失败，不影响核心格式。
- 图片超大：按屏幕适应显示，必要时提示内存不足。

## 性能策略

- 原生 Win32，无托管运行时依赖。
- 主程序启动路径不加载扩展 DLL。
- WIC 解码走系统组件。
- Direct2D/DXGI 硬件加速绘制。
- 后台预读前后各 1 张，受内存限制。
- 大图缩放优先使用解码阶段缩略能力；必要时分阶段渲染，避免 UI 卡死。
- UI 消息循环不执行长时间解码和下载任务。
