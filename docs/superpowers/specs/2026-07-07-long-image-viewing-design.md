# cpictures 长图/宽图浏览修复设计

## 目标

修复长图或宽图打开显示异常、内容溢出不可浏览、从长图切换到其他图片时报 `startup failed` 的问题。新增极简浏览交互：按住 `Shift` 后鼠标变手型，左键拖动浏览超出窗口的内容；键盘 `↑` / `↓` 也可滚动长图或宽图。

## 当前问题判断

现有逻辑会把窗口尺寸限制到屏幕工作区内，但渲染仍按 `viewState.zoom == 1.0` 把原图居中绘制。长图或宽图被裁切后没有 pan offset，用户无法浏览被裁掉区域。切换图片时旧渲染状态、窗口 resize 和 D2D bitmap 重建可能在长图尺寸下触发异常，异常冒到主消息循环后显示 `cpictures: startup failed.`。

## 显示规则

### 竖长图

定义：图片高度大于屏幕工作区高度，且图片宽度不大于屏幕工作区宽度。

- 水平方向按 100% 原图显示。
- 窗口宽度贴合图片宽度。
- 窗口高度最多等于屏幕工作区高度。
- 初始显示图片顶部。
- 允许上下 pan。

### 横宽图

定义：图片宽度大于屏幕工作区宽度，且图片高度不大于屏幕工作区高度。

- 垂直方向按 100% 原图显示。
- 窗口高度贴合图片高度。
- 窗口宽度最多等于屏幕工作区宽度。
- 初始显示图片左侧。
- 允许左右 pan。

### 双向超大图

定义：图片宽度和高度都超过屏幕工作区。

- 窗口宽度最多等于屏幕工作区宽度。
- 窗口高度最多等于屏幕工作区高度。
- 仍按 100% 原图显示。
- 初始显示左上角。
- 允许水平和垂直 pan。

### 普通图片

- 继续使用当前默认 100% 原图、窗口贴合图片尺寸、上限不超过屏幕工作区。
- 无可 pan 内容时，左键拖动仍移动窗口。

## 交互规则

- 按住 `Shift` 且当前图片有可 pan 内容时，鼠标指针切换为手型。
- 按住 `Shift` + 鼠标左键拖动：移动图片内容，不移动窗口。
- 不按 `Shift`：鼠标左键保持现有窗口拖动行为。
- `↑`：如果是竖长图，内容向上滚动；如果是横宽图，内容向左滚动。
- `↓`：如果是竖长图，内容向下滚动；如果是横宽图，内容向右滚动。
- 切换图片、重新加载图片、切换实际大小/适应屏幕时，pan offset 重置到初始位置。
- 鼠标滚轮继续保持当前前后切图行为，不改为滚动长图。

## 渲染规则

新增 pan offset 到 `ViewState`。D2D 渲染时图片目的矩形不再固定居中，而是根据：

- 当前缩放后的图片尺寸。
- 当前窗口 client 尺寸。
- pan offset。

普通图片仍居中。超出窗口的方向按 pan offset 偏移，且 offset 被 clamp 到有效范围，不能拖出空白边。

## 状态边界

- 可 pan 判断来自：缩放后图片尺寸是否大于 client 尺寸。
- 竖向可 pan：`drawHeight > clientHeight`。
- 横向可 pan：`drawWidth > clientWidth`。
- pan 范围：
  - `0 <= panX <= drawWidth - clientWidth`
  - `0 <= panY <= drawHeight - clientHeight`
- 渲染目的矩形：
  - 如果该方向可 pan，起点为 `-pan`。
  - 如果该方向不可 pan，仍居中。

## 测试策略

- 单元测试窗口尺寸：
  - 竖长图 `800x4000`、工作区 `1920x1080` -> 窗口 `800x1080`。
  - 横宽图 `4000x800`、工作区 `1920x1080` -> 窗口 `1920x800`。
  - 双向超大图 `4000x3000`、工作区 `1920x1080` -> 窗口 `1920x1080`。
  - 小图仍保持原尺寸。
- 单元测试 pan clamp：
  - 超出内容时 offset 可变。
  - offset 小于 0 时归 0。
  - offset 超过最大值时归最大值。
- 手动 smoke：
  - 打开竖长测试图，窗口高度不超过屏幕，高度方向可 Shift 拖动。
  - 打开横宽测试图，窗口宽度不超过屏幕，宽度方向可 Shift 拖动。
  - 长图和普通图之间前后切换不报 `startup failed`。

## 非目标

- 不修改鼠标滚轮前后切图行为。
- 不添加滚动条。
- 不添加可见工具栏或状态栏。
- 不改变右键菜单结构。
